About Decidr

In today's fast-paced world, groups of people often struggle with the challenge of making decisions efficiently. Whether it's choosing a restaurant for a family dinner, selecting a movie to watch with friends, determining the best football team, or deciding on a game to play, indecisiveness can lead to wasted time and frustration. Current methods of resolving disagreements and reaching a consensus can be cumbersome and time-consuming, often resulting in unresolved disputes. As a result, there is a pressing need for a user-friendly, efficient solution that empowers groups of all ages and demographics to make timely, informed decisions in a manner that fosters collaboration and unity.
Decidr, our innovative web-based application offers an elegant and user-friendly solution to this common problem of indecisiveness within groups. By enabling an appointed admin to initiate a decision-making session and invite participants with a unique code, Decidr streamlines the decision-making process. Each user contributes a limited number of suggestions relevant to the topic at hand and the application then intelligently aggregates and displays the suggestions as bubbles on the screen, merging similar ideas into larger, proportionally representative bubbles. Our primary goal is to minimize the time and stress associated with deciding on various topics, offering multiple preferred options that truly reflect the group's collective preferences. Decidr is an innovative solution to the age-old problem of group decision-making, providing an effective and engaging platform for making choices together.

User Requirements:

User Type 1: Admin
On a hot summer day, Aisha wants to go out for ice cream with her friends, but they can’t make a decision on where to go since they keep shouting over each other. Frustrated, she pulls out her phone and opens Decidr.
In Decidr she creates a decision-making session titled “Something Cold Now”, and sends out invites to her friends. As her friends join her session, Aisha becomes the admin. She can now set rules, time limits, etc. for the session in order to come to a timely and uniform decision.

User Type 2: Members
While trying to help Aisha make a decision, all of her friends got extremely passionate and started talking over each other. As a result, Aisha resorts to Decidr to resolve the conflict.
Once they receive a unique code from Aisha, all her friends join her session as members. Here they can collaborate in a more productive manner. Based on the rules and restrictions Aisha sets, the members can now vote in a silent and effective manner. Some of the things they can do include adding or making new suggestions and collaborating with each other. In the end, they all come to a collective decision.

Key Features:

As we go through the project we will be creating some key features with set timelines. We will be going through version control such as Git and participate as a group where each member partakes in their own feature to implement onto the product. Our plan is to create new feature branches through our local repository and then do a pull request before merging onto our main branch after we have completed our feature branch.

  • Login Screen: Users will be prompted two fields, an input field to enter the code to join an existing meeting or a button to begin a new session. Upon creating the meeting room, the code will be displayed on the screen. Existing members in the room can share this code to invite participants to the application.
  • Admin Screen: Upon creating a meeting room, Admins can provide questions and suggestions, then prompt participants for votes. The user interface will display a circle. The size of that circle corresponds with the number of votes that have accumulated for that suggestion. Once the votes are collected, admins are given a clear visual representation of the most popular suggestion. Admins are also given the task of extending the time for votes, ending a vote, deleting suggestions, editing votes, or adding more votes.
  • Member Screen: Members are given an interface similar to the Admin, but are given fewer privileges as they are limited to just providing votes or adding suggestions which would then have to be approved by the admin.
  • Result Screen: Displayed for all users. Once the votes have rallied up, a table will order the options in decreasing order from highest vote counts to lower vote counts. A button will also prompt the users to return back to the home page.
Design Features:
  1. Collaborative Decision-Making: Decide facilitates collaborative decision-making by allowing users to contribute their suggestions, view other’s ideas, and collectively shape the outcome.
  2. Real-Time Interaction: The application operates in real-time enabling users to see the dynamic changes. Users can make additional suggestions during the decision-making process.
  3. Anonymous Voting and Results: Decide tallies votes anonymously and present the top options at the end of a preset time limit, facilitating a quick and democratic decision-making process.
  4. Admin Control: The admin plays a pivotal role in initiating decision-making sessions, sending unique codes to invite participants, and setting parameters for the session, ensuring a guided and organized process.
  5. Responsive UI with React: The front end will be built using React to provide an interactive and user-friendly interface, handling rendering, state management, etc. We will also optimize the pages so that they cater to all screen sizes to ensure a large array of participants are able to effectively participate.
  6. MongoDB Data Storage: The back end will be built using MongoDB which stores and retrieves data related to user questions, choices, votes, suggestions, etc. This will ensure that historical suggestions are created. The back end will be created later in the timeline once the front end is fully implemented.
